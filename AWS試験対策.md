# AWS 試験対策


## ややこしいもの
### セキュリティグループとネットワークACL

|名前|制御単位|設定特徴|
|---|---|---|
|セキュリティグループ|インスタンス|ステートフル, 同じサブネット内のみデフォルトでやり取り, 許可ルールのみ指定|
|ネットワークACL|VPC/サブネット|ステートレス(IN/OUTが同じ), デフォルトですべてオープン|



### ログ・セキュリティ・モニタリング

|名前|役割|
|---|---|
|AWS Config |AWS **リソース**の設定を評価、監査、審査|
|Amazon Inspector|自動化されたセキュリティ**評価**サービス|
|Amazon CloudWatch |**アプリケーションを監視**,リソース使用率の最適化|
|AWS CloudTrail |AWS インフラストラクチャ全体で**アカウントアクティビティ**をログに記録|



|名前|役割|
|---|---|
|AWS Shield |**DDoS攻撃**に対する保護サービス|
|AWS WAF|**SQL インジェクション**やクロスサイトスクリプティングなど 一般的な Web エクスプロイト（攻撃）から Web アプリケーションを保護, <br>CloudFront ディストリビューションまたは Application Load Balancer でリクエスト(IPアドレス)をフィルタリングおよびブロック|
|GuardDuty|悪意のあるアクティビティと不正な動作を監視する**脅威検出**サービス, 保護はしない|


### 場所

|名前|役割|
|---|---|
|AWS Local Zones|ローカルで動作するアプリケーションへの非常に低いレイテンシーのアクセスを提供|
|AWS エッジロケーション|CloudFront がコンテンツのコピーをキャッシュするために使用するサイト（拠点）|
|Global Accelerator|**グローバルネットワーク**を利用, TCP/UDPを介したトラフィックのパフォーマンスを最適化, エッジから、1 つまたは複数のリージョンで実行されるアプリケーションへのパケットをプロキシする|
|CloudFront |**エッジロケーション**を利用, 低レイテンシーの高速転送により世界中の視聴者に安全に配信する高速コンテンツ配信ネットワーク（CDN）サービス|


### ストレージ
|名前|キーワード|
|---|---|
|EC2 インスタンスストア|一時的な**ブロック**レベルのストレージ|
|Elastic File System (EFS)|NFS ファイルシステム, 大規模な並列共有アクセスを提供, 自動で拡大および縮小|
|Elastic Block Store (EBS)|ブロック, どんな規模のワークロードにも対応, 一度に** 1 つの EC2 インスタンス**にしか実装できない|
|S3|オブジェクト, 様々なケースのデータを任意の量で保存および保護,インスタンス間のファイル共有には使用できない|


### データベース
|名前|キーワード|
|---|---|
|Amazon DynamoDB|**NoSQL, ドキュメントデータ, サーバーレス, フルマネージド、マルチリージョン、マルチマスター**|
|Amazon Redshift |大規模, フルマネージド型, データウェアハウス|
|Amazon Aurora|**リレーショナルデータベース**|

### 自動化、コンテナ
|名前|役割|
|---|---|
|AWS Fargate|**コンテナ向け**サーバーレスコンピューティングエンジン, **ECSとEKS上で**動作|
|AWS Step Function|複数の AWS サービスを**サーバーレスワークフロー**に調整|
|AWS CloudHSM|AWS クラウドで**暗号化キー**を簡単に生成, ハードウェアのプロビジョニング、ソフトウェアへのパッチ適用、高可用性、バックアップといった時間のかかる管理タスクを自動化|
|AWS OpsWorks |コードを使用して**サーバーの構成を自動化**するインスタンスを利用|
|AWS CloudFormation |すべてのリージョンおよびアカウントにまたがって、アプリケーションに必要なすべてのリソースを、自動化された安全な方法で、**プロビジョニング**, プログラミング言語やシンプルなテキストファイル（YAML や JSON 形式）を使用|
|AWS Elastic Beanstalk |Web アプリケーションやサービスを**自動的にデプロイおよびスケーリング**するための、簡単に使えるサービス|
|Amazon Elastic Container Registry (ECR)｜**Docker コンテナイメージを保存、管理、およびデプロイ**|
|Amazon Elastic Container Service (ECS) |クラスター上で **Docker コンテナを簡単に実行、停止、管理**できる拡張性が高く高速なコンテナ管理サービス|

## よく問われるもの
- グローバルサービス: **AWS IAM、Amazon CloudFront、Route 53、WAF**
- オンプレミスネットワークを AWS クラウドに接続するために使うサービス: VPN, Direct Connect
- アウトバウンドデータ転送のみが課金
- Route53の特徴: 
    - 地理的なルーティングポリシーを使用して、特定の地域をブロック
    - コンテンツをローカライズして、Web サイトの一部またはすべてをユーザーの言語で表示できる
